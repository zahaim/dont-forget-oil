# Project Idea: Fuel & Maintenance Tracker

## Vision

A privacy-first mobile app that replaces manual fuel consumption tracking with simple photo-based logging. Users take two photos during refueling (car mileage and fuel pump meter), and the app automatically extracts the data to track fuel consumption, costs, and maintenance schedules.

## Problem Solved

Currently tracking fuel consumption and car maintenance requires:
- Manual spreadsheet updates (Google Drive, Excel)
- Recording odometer and pump readings by hand
- No automated reminders for service tasks
- Data scattered across multiple documents

This app consolidates fuel and maintenance tracking into one simple, visual tool.

## Core Motivation

- **Personal use**: Replace a manual Google Drive spreadsheet currently maintained
- **Privacy-first**: No cloud storage, no personal data collection, no ads
- **Open-source**: Built as a helper tool, not a commercial product
- **User control**: All data stored as simple, portable flat files

## Target Users (Initial)

- Car owners who actively track fuel consumption
- Privacy-conscious users who distrust cloud services
- People interested in vehicle maintenance visibility
- Potentially: mechanics, fleet managers (future iterations)

## App Scope - Three Phases

### Phase 1: Fuel Tracking (MVP)
- Photo-based fuel logging (odometer + pump meter)
- Cost tracking per refuel
- Basic data visualization (fuel price trends, monthly costs)
- Replaces existing manual spreadsheet

### Phase 2: Maintenance Reminders
- Car database or manual entry of service intervals
- Oil change reminders (based on mileage or time)
- Filter replacement reminders
- General service task scheduling (brakes, liquids, windshields, etc.)

### Phase 3: Seasonal Reminders
- Winter/summer tire change notifications
- Seasonal maintenance checklists

## Key Principles

1. **Privacy**: No personal data stored. Photos deleted immediately after processing.
2. **Portability**: Data in simple flat files or single text database, shareable via any cloud drive
3. **Simplicity**: Easy to use for non-technical users
4. **Open-source**: Licensed appropriately, source code public
5. **Offline-first**: Works without internet connection
6. **User ownership**: Users control their data completely
